# QNN Financial Statement Analysis

Master's thesis research project on Quantum Neural Networks for financial statement analysis.

## Project scope

This repository supports research on the use of Quantum Neural Networks and classical machine learning models for analyzing financial statement data.

The project may include:

- financial statement preprocessing,
- financial ratio engineering,
- fraud or bankruptcy proxy construction,
- classical machine learning baselines,
- Quantum Neural Network experiments,
- model comparison and evaluation,
- reproducible notebooks and reports.

## Repository structure

```text
data/          datasets used in the empirical part of the thesis
docs/          research notes, methodological assumptions, and project documentation
notebooks/     exploratory notebooks for data analysis and QNN/ML experiments
references/    bibliography, literature notes, and source summaries
reports/       generated experiment reports, tables, plots, and model evaluation results
src/           reusable Python modules for preprocessing, feature engineering, modelling, and evaluation
thesis/        thesis drafts, chapter materials, and final written outputs
```

## Current Data Pipeline

The reproducible SEC preprocessing flow is organized as numbered steps in
`src/data/`.

For the current modeling dataset stage:

```bash
python src/data/08_build_modeling_dataset.py
```

This builds the first ML/QNN-ready company-year dataset from parsed SEC facts,
sanity-check warnings, and `configs/dataset_config.yaml`.

The numbered script is a thin pipeline runner. The reusable deterministic
builder lives in `src/data/modeling_dataset.py`, while the readable EDA and
methodology narrative lives in the notebook.

Main Commit 08 artifacts:

- `notebooks/08_modeling_dataset_eda.ipynb` - exploratory notebook for SGH thesis pre-experimental analysis,
- `data/processed/modeling_dataset.csv` - main dataset for later modeling,
- `data/processed/modeling_dataset_excluded.csv` - excluded observations with explicit reasons,
- `data/reports/modeling_dataset_quality_report.md` - technical dataset quality documentation,
- `reports/figures/commit_8/` - diagnostic figures generated by the notebook.

Commit 08 does not train models, scale features, impute missing values, or run
QNN code. It prepares auditable inputs for the next experimental stages.
